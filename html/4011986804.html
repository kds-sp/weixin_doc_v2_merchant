APP纯签约

更新时间：2025.09.24

外部App拉起微信客户端发起签约前，需先后台调用预签约接口完成预签约，获取pre_entrustweb_id，再拉起微信客户端，完成签约，返回App。

注意：

  • 该业务接口暂未全面开放，需要单独申请后方可使用，详情可查看申请指引：委托代扣-App&H5纯签约申请流程

步骤1：预签约接口

接口说明

适用对象：直连商户

请求URL： https://api.mch.weixin.qq.com/papay/preentrustweb

请求方式： POST

数据格式： XML

签名方式： HMAC-SHA256、MD5

是否需要证书： 否

请求参数

参数名

变量

类型[长度限制]

必填

描述

应用ID

appid

string[1,32]

是

直连商户申请的公众号或移动应用appid。 （请登录开放平台查看，注意与公众号的APPID不同）
示例值：wxcbda96de0b165486

商户号

mch_id

string[1,32]

是

商户号是商户在微信申请微信支付成功后分配的账号ID，登录平台为pay.weixin.qq.com
示例值：1200009811

模板id

plan_id

string[1,28]

是

协议模板id，设置路径见开发步骤。
示例值：12535

签约协议号

contract_code

string[1,128]

是

商户侧的签约协议号，由商户生成，只能是数字、大小写字母的描述。
示例值：100000

请求序列号

request_serial

int 64

是

商户请求签约时的序列号，要求唯一性。禁止使用0开头，序列号主要用于排序，不作为查询条件，纯数字，范围不能超过int64的范围（9223372036854775807）。
示例值：1000

用户账户展示名称

contract_display_account

string[1,128]

是

签约用户的名称，用于用户签约页面的“开通账号”处进行展示，参数值不支持UTF8非3字节编码的字符，例如表情符号，所以请勿传微信昵称到该字段
示例值：微信代扣

回调通知url

notify_url

string[1,256]

是

用于接收签约成功消息的回调通知地址，以http或https开头，通知url必须为外网可访问的url，不能携带参数。

版本号

version

string[1,8]

是

固定值1.0
示例值：1.0

签名类型

sign_type

string[1,32]

否

MD5或者HMAC-SHA256，不填默认MD5
注意：app预签约的默认签名方式是(sign_type为空或无该字段)：MD5
示例值：MD5

签名

sign

string[1,64]

是

详见签名生成算法
示例值：C380BEC2BFD727A4B6845133519F3AD6

时间戳

timestamp

string[1,10]

是

系统当前时间，10位
示例值：1414488825

返回app

return_app

string[1,1]

否

用来控制签约页面结束后的返回路径（不传此参数，则签约完成后停留在微信内）。
Y表示返回app, 不填则不返回 注：签约参数appid必须为发起签约的app所有，且在微信开放平台注册过。
ios和android用户点完成按钮才返回原APP，直接右划关闭页面或左上角不支持返回APP
示例值：Y

请求示例：

XML

1<xml>
2 <appid>wxcbda96de0b165486</appid>
3 <mch_id>1200009811</mch_id>
4 <plan_id>12535</plan_id>
5 <contract_code>100000</contract_code>
6 <request_serial>1000</request_serial>
7 <contract_display_account>微信代扣</contract_display_account>
8 <notify_url>https://weixin.qq.com</notify_url>
9 <version>1.0</version>
10 <sign_type>MD5</sign_type>
11 <sign>C380BEC2BFD727A4B6845133519F3AD6</sign>
12 <timestamp>1414488825</timestamp>
13 <return_app>Y</return_app>
14</xml>

跳转签约小程序的username

miniprogram_username

string[1,32]

否

只有以下两种情况才会返回该参数：
1、模板ID为2025年9月23日及之后申请的委托代扣模板
2、模板ID为2025年9月23日之前申请的委托代扣模板，但申请了WXLaunchMiniProgram权限

若返回该字段，则可使用LaunchMiniProgram SDK进行签约，详情可参考APP调起签约，需要将该字段名和值原样填入；若未返回该字段，说明该模版只能通过OpenBusinessWebview SDK进行签约
示例值：gh_xxxxxxxxxxxxx

步骤2：签约接口

接入流程

一、该接口需要额外申请OpenBusinessWebview权限（仅支持在2025年9月22日之前申请过OpenBusinessView权限的商户在存量模板使用。2025年9月23日起，不再开放申请，且新增模板不可使用OpenBusinessView。请参考APP调起签约-WXLaunchMiniProgram文档，使用WXLaunchMiniProgram，用户体验更好。），请联系对接您的运营同学协助申请，具体参看接入流程

     邮件模板如下：

     标题：OpenBusinessWebview权限申请_APP纯签约

     正文：

      （1）说明使用背景

      （2）appid、mchid（appid和mchid需要有绑定关系）

二、在微信开放平台（open.weixin.qq.com）注册APP信息

三、在开放平台-资源中心下载SDK：资源下载

四、按照下面的接口指引文档完成对接

接口说明

适用对象：直连商户 服务商

请求方式： POST

请求参数

参数名

变量

类型[长度限制]

必填

描述

预签约id

pre_entrustweb_id

string[1,128]

是

预签约id
示例值：5778aadY9nltAsZzXixCkFIGYnV2V

注意：

预签约id（pre_entrustweb_id）需要从预签约接口的返回参数获取，其他参数请严格按照下方接入代码示例传入

接入代码示例：

IOS

1WXOpenBusinessWebViewReq *req = [[WXOpenBusinessWebViewReq alloc] init];
2req.businessType =12; //固定值
3NSMutableDictionary *queryInfoDic = [NSMutableDictionary dictionary];
4[queryInfoDic setObject:"5778aadY9nltAsZzXixCkFIGYnV2V" forKey:"pre_entrustweb_id"];
5req.queryInfoDic = queryInfoDic;
6[WxApi sendReq:req];

Android

1WXOpenBusinessWebview.Req req = new WXOpenBusinessWebview.Req();
2req.businessType = 12;//固定值
3HashMap  queryInfo = new HashMap<>();
4queryInfo.put("pre_entrustweb_id","5778aadY9nltAsZzXixCkFIGYnVV1");
5req.queryInfo = queryInfo;
6api.sendReq(req);

返回参数

WXOpenBusinessWebview.Resp，返回参数内容无需关注，如果签约成功，商户系统会收到“签约结果通知”
